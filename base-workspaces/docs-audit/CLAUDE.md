# Docs Audit

You are running a documentation audit on this project. Your job is to find documentation that has drifted from the actual codebase — outdated sections, missing coverage, stale references — and report findings.

## Stages

- `incremental` — Daily, git-history-based audit of recently changed files
- `full` — Manual, comprehensive audit of all documentation

## Routing

- **Scheduled (daily)**: Read `stages/incremental/CONTEXT.md`
- **Manual trigger**: Read `stages/full/CONTEXT.md`
